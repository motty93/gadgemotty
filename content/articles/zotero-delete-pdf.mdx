Zoteroではエントリまたはカテゴリを削除しても、その添付ファイル（PDF・リンクされた添付ファイルを含む）は削除されません。したがってエントリの追加と削除を繰り返すと無駄なPDFファイルがどんどん溜まっていくことになってしまいます。これを解決する方法を紹介します。

## 問題点

Zoteroは優れた文献管理ソフトウェアですが、エントリを削除しても添付ファイルが自動的に削除されないという仕様があります。これにより、以下のような問題が発生します：

* ストレージ容量の無駄遣い
* 不要なファイルの蓄積
* 同期時間の増加

## 解決方法

この問題を解決するには、以下の方法があります：

### 1. Zotero File Cleaner プラグインを使用する

Zotero File Cleaner は、未使用の添付ファイルを検出して削除するためのプラグインです。

1. Zotero を開き、ツール → アドオン を選択します。
2. 「プラグインを検索」欄に「Zotero File Cleaner」と入力します。
3. プラグインをインストールし、Zotero を再起動します。
4. ツール → Zotero File Cleaner を選択して実行します。

### 2. 手動でストレージディレクトリをクリーンアップする

Zotero のストレージディレクトリを手動でクリーンアップすることもできます：

1. Zotero を閉じます。
2. Zotero のデータディレクトリを開きます（通常は「ツール → 環境設定 → 詳細 → データディレクトリの場所を表示」で確認できます）。
3. 「storage」フォルダ内の未使用ファイルを削除します。

ただし、この方法は危険性が高いため、事前にバックアップを取ることをお勧めします。

### 3. Zotero の設定を変更する

Zotero 5.0以降では、エントリを削除する際に添付ファイルも削除するように設定できます：

1. Zotero を開き、編集 → 環境設定 を選択します。
2. 「詳細」タブを選択します。
3. 「アイテムを削除する際に添付ファイルも削除する」オプションをチェックします。
4. 「OK」をクリックして設定を保存します。

この設定を有効にすると、今後エントリを削除する際に添付ファイルも自動的に削除されます。

## まとめ

Zotero でエントリを削除する際に添付ファイルも削除するには、プラグインを使用するか、Zotero の設定を変更するのが最も安全で効果的な方法です。これにより、ストレージ容量を節約し、Zotero の動作を最適化することができます。
